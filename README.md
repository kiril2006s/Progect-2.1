# Прогноз опадів за допомогою машинного навчання

## Опис проєкту

Даний проєкт реалізує систему прогнозування опадів на основі історичних метеорологічних даних.
Для отримання погодних даних використовується API сервісу **Open-Meteo**, після чого дані обробляються та використовуються для навчання моделі машинного навчання.

Метою проєкту є створення програми, яка:

* отримує погодні дані за заданий період;
* формує датасет у форматі CSV;
* навчає модель класифікації;
* прогнозує наявність опадів;
* відображає результат у веб-інтерфейсі.

Інтерфейс реалізований за допомогою Streamlit, що дозволяє запускати програму як простий веб-застосунок.

---

# Використані технології

У проєкті використано такі технології та бібліотеки:

* Python — основна мова програмування
* pandas — обробка та аналіз даних
* requests — отримання даних з API
* scikit-learn — реалізація алгоритмів машинного навчання
* Streamlit — створення веб-інтерфейсу

---

# Джерело даних

Метеорологічні дані отримуються через API сервісу Open-Meteo.

Використовуються щоденні (daily) погодні показники, зокрема:

* temperature_2m_max — максимальна температура за день
* temperature_2m_min — мінімальна температура за день
* precipitation_sum — сумарна кількість опадів
* rain_sum — сумарна кількість дощу
* wind_speed_10m_max — максимальна швидкість вітру

Отримані дані зберігаються у файл:

weather_daily.csv

---

# Формування цільової змінної

Для задачі класифікації використовується бінарна цільова змінна (label):

0 — опадів немає
1 — опади є

Label формується на основі параметра precipitation_sum:

* precipitation_sum = 0 → label = 0
* precipitation_sum > 0 → label = 1

Таким чином модель навчається визначати, чи будуть опади.

---

# Алгоритм машинного навчання

Для класифікації використовується алгоритм Logistic Regression.

Процес навчання моделі включає:

1. Формування матриці ознак (X)
2. Формування цільової змінної (y)
3. Розбиття даних на навчальну та тестову вибірки
4. Навчання моделі
5. Оцінку точності класифікації

Для оцінки моделі використовуються метрики:

* Accuracy
* Precision
* Recall
* F1-score

# Функціональність програми

Застосунок має наступні функції:

## 1. Отримання даних

Користувач може отримати погодні дані з Open-Meteo, задавши:

* координати (latitude, longitude)
* дату початку
* дату кінця

Після цього дані автоматично зберігаються у CSV файл.

---

## 2. Завантаження датасету

Користувач може завантажити файл weather_daily.csv у програму для подальшого аналізу.

---

## 3. Навчання моделі

Після завантаження даних користувач може натиснути кнопку:

"Навчити модель"

Програма виконує навчання моделі та відображає метрики точності.

---

## 4. Прогноз опадів

Після навчання моделі можна виконати прогноз для конкретного дня з датасету.

Результат прогнозу відображається у вигляді повідомлення:

"Очікуються опади"

або

"Опадів не очікується"

Також виводиться ймовірність того, що в цей день будуть опади.

---

# Висновок

У рамках даного проєкту була реалізована система прогнозування опадів на основі історичних метеорологічних даних.
Було виконано отримання даних через API, створено датасет, реалізовано модель машинного навчання та веб-інтерфейс для взаємодії з користувачем.
Отримана модель дозволяє оцінювати ймовірність появи опадів на основі погодних параметрів, що демонструє можливості застосування методів машинного навчання для аналізу метеорологічних даних.
